# 设计系统（Design System）

本文件定义 Quant Arb Console 的视觉规范。整体风格为**深色专业交易后台**。  
**参考基准**：三张 UI 截图（Dashboard 概览、套利机会发现、资金费套利任务列表）。

---

## 1. 设计原则

1. **信息密度优先**：交易后台需在有限屏幕内展示大量数据，组件紧凑，留白克制
2. **一致性**：相同数据类型在任何页面使用相同的颜色、格式、交互
3. **可扫描性**：关键指标通过颜色/大小/位置快速识别，减少阅读成本
4. **克制动效**：仅在状态变化时使用短暂动效（200-300ms），不干扰数据阅读

### 2.6 颜色使用规则

- **收益/亏损**：所有金额和百分比变化统一使用 `--color-profit`（正值）和 `--color-loss`（负值），零值使用 `--text-secondary`
- **风险等级**：低风险=profit（绿色），中风险=warning（橙色），高风险=loss（红色）
- **状态点**：运行中=绿色实心圆 `●`，已完成=灰色实心圆 `●`，异常=红色实心圆 `●`

## 3. 字体与排版

### 3.1 字体栈

```css
--font-sans: 'Inter', -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif;
--font-mono: 'JetBrains Mono', 'Fira Code', 'Consolas', monospace;
```

- 界面文字使用 `--font-sans`
- 数字、金额、币种代码、哈希 ID 使用 `--font-mono`
- **数字必须使用等宽字体**，保证表格列对齐

### 3.2 字号层级

| 层级         | 大小     | 行高   | 字重   | 用途                                           |
|-------------|----------|--------|--------|------------------------------------------------|
| Display     | 32px     | 40px   | 700    | Dashboard 大数字（如 1,245,678.32）             |
| H1          | 20px     | 28px   | 600    | 页面标题（如"量化交易系统"、"套利机会发现"）     |
| H2          | 16px     | 24px   | 600    | 卡片标题（如"资产总览"、"收益分析"）             |
| H3          | 14px     | 20px   | 600    | 子区块标题、KPI 标签                            |
| Body        | 14px     | 20px   | 400    | 正文、表格内容                                 |
| Caption     | 12px     | 16px   | 400    | 辅助说明、时间戳、脚注、侧边栏图标下方文字      |
| Micro       | 11px     | 14px   | 500    | Badge、Tag、状态点旁文字                        |

### 3.3 数字格式化

| 数据类型        | 格式                     | 示例                         |
|----------------|--------------------------|------------------------------|
| USDT 金额       | 千分位 + 4 位小数         | `1,245,678.32 USDT`         |
| USDT 金额（短） | 千分位 + 2 位小数         | `+12,345.67 USDT`           |
| 法币换算        | `≈ ¥` + 千分位 + 2 位    | `≈ ¥8,916,549.72`           |
| 百分比          | 带符号 + 2 位小数 + `%`  | `+24.53%`, `-1.25%`         |
| 资金费率        | 带符号 + 4 位小数 + `%`  | `+0.0432%`, `-0.0131%`      |
| 年化收益率      | 带符号 + 2 位小数 + `%`  | `+21.78%`, `-35.12%`        |
| 持仓数量        | 4 位小数                  | `200.0000`                   |
| 合约张数        | 整数 + ` 张`              | `207 张`                     |
| 倒计时          | `HH:mm:ss`               | `02:12:34`                   |
| 时间            | `YYYY-MM-DD HH:mm:ss`    | `2026-04-25 14:30:00`        |
| 杠杆倍数        | `(Nx)`                   | `(3x)`                      |

## 5. 通用组件规范

### 5.1 按钮

| 类型       | 样式                                       | 场景                             |
|-----------|-------------------------------------------|----------------------------------|
| Primary   | 蓝色填充 `--color-primary`                 | 添加套利、确认操作                |
| Success   | 绿色填充/边框 `--color-profit`             | 金额按钮（如截图中 `100 USDT`）  |
| Danger    | 红色填充 `--color-loss`                    | 停止任务、平仓、删除              |
| Default   | 深色背景 + 浅色边框                         | 普通操作、筛选、重置              |
| Ghost     | 无边框透明                                  | 工具栏按钮、次要操作              |
| Text      | 纯文字绿色/蓝色                             | 链接式操作（如"添加套利"、"详情"）|

截图中特殊按钮：
- **金额操作按钮**（截图1）：绿色边框 + 绿色文字，如 `100 USDT`，表示可操作的资金分配
- **Closed 按钮**（截图1）：灰色背景，表示已关闭仓位的不可操作状态
- **添加套利**（截图3）：绿色文字链接按钮，表格行内操作
- **底部大按钮**（截图3详情面板）：全宽蓝色/绿色填充按钮 `⊕ 添加套利`

### 5.2 表格

- 行高 `48px`（默认）
- 表头背景 `--bg-card`，字号 Caption，颜色 `--text-secondary`
- 行 hover 高亮 `--bg-row-hover`
- **选中行**（截图3）：浅蓝色背景高亮 `--bg-active`

## 6. 响应式策略

| 断点             | 宽度范围          | 策略                                     |
|-----------------|-------------------|------------------------------------------|
| Desktop XL      | ≥ 1440px          | 完整布局，分栏详情面板可展开              |
| Desktop         | 1024–1439px       | 完整布局，详情面板压缩或变为覆盖式抽屉    |
| Tablet          | 768–1023px        | 侧边栏隐藏为汉堡菜单，详情面板全屏       |
| Mobile          | < 768px           | MVP 不做，仅保证基本可读                 |

**桌面端优先**：设计稿和开发以 1440px 为基准宽度。
