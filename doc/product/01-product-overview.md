# 产品概览（Product Overview）

## 1. 产品定义

Quant Arb Console 是一套 **桌面端优先** 的专业量化交易后台，核心聚焦“**多交易所资金费率套利**”，并逐步扩展到策略监控、资产分析、风控告警、历史记录与回测。

它的定位更接近“量化交易控制台”，而不是普通看盘软件或散户用的交易终端。[file:1][file:2][file:3]

---

## 2. 核心目标

1. **发现套利机会**  
   实时扫描多家交易所（Binance、Bybit、OKX 等）的资金费率和合约信息，识别同一币种、可对冲的资金费率套利机会。

2. **评估可执行性**  
   不仅看费率差，还要综合盘口深度、手续费、滑点、API 状态、账户资金、风险规则，给出“可执行性评分”。

3. **创建套利任务**  
   通过套利机会页一键创建任务，默认采用“人工确认后执行”的半自动模式，支持配置目标仓位、使用账号等。

4. **监控执行与收益**  
   跟踪任务生命周期：订单下发、成交、持仓、结算、平仓，展示预估收益 vs 实际收益、扣费后净收益、偏差情况。[file:3]

5. **集中风控和审计**  
   统一管理杠杆、敞口、单所集中度、API 健康、异常订单/任务，并记录所有关键操作的审计日志，方便复盘和合规。

---

## 3. MVP 范围

### 3.1 包含内容（必须有）

- 资金费率套利机会扫描（多交易所、永续合约为主）
- 统一合约标识和基本标准化（symbol 对齐、精度、最小下单量）
- 套利机会列表 + 详情页（类似你截图里的“套利机会发现”页）[file:1]
- 创建套利任务（半自动：人工确认后执行）
- 订单与仓位跟踪（基本实时）
- 基础风控：账户资金检查、杠杆限制、单所集中度、API 健康
- 收益统计：预估收益 / 实际收益 / 扣费后净收益
- Dashboard 总览（资产、收益、策略运行、风险概览）[file:2]
- 简单告警：API 异常、任务异常、风控触发

### 3.2 不包含内容（后续版本再做）

- 完全无人值守的全自动执行
- 高频 / 低延迟套利（撮合机房部署等）
- 跨所现货–合约价差套利、趋势策略、网格策略、统计套利等其他策略
- 完整回测与参数优化平台
- 多租户 SaaS 化、细粒度计费
- 完整移动端应用（只做基本适配）

## 4. 业务边界

### 4.1 本系统不是

- 不是交易所本身，不负责撮合撮单。
- 不是通用行情终端，不以“看 K 线”为主。
- 不是全功能量化平台（回测/研究/撮合一体）。
- 不是资金托管或出入金系统。

### 4.2 本系统是

- 针对“资金费率套利”这类 **Delta 近中性的策略** 的机会发现和执行控制台。
- 把“机会计算 / 风控 / 执行 / 收益 / 审计”整合在一起的运营后台。
- 后续可以平滑扩展到其他套利或中低频策略的统一监控平台。

## 5. 用户角色

| 角色           | 主要诉求                               | 典型操作路径                                             |
|----------------|----------------------------------------|----------------------------------------------------------|
| 超级管理员     | 配置系统、控制权限、保证安全与合规     | 配置交易所账户 → 配置风控规则 → 管理角色与权限          |
| 交易员         | 稳定跑策略、看机会、看收益             | 看 Dashboard → 看套利机会 → 创建任务 → 监控任务          |
| 风控员         | 控敞口、控杠杆、控集中度、处理异常     | 看 Dashboard 风控概览 → 风控中心 → 处理告警 → 调规则     |
| 观察员 / 合伙人 | 快速了解整体表现，不改配置             | 看 Dashboard → 看历史和分析 → 导出报告                   |

## 6. 关键设计约束

1. **桌面端优先 / 深色专业风格**  
   参考你提供的三张 UI 截图：左侧导航、顶部 KPI、中间高密度表格/图表、右侧详情抽屉。主色系以深蓝、深灰为主，绿色表示收益为正，红色表示风险或亏损。[file:1][file:2][file:3]

2. **先半自动后全自动**  
   MVP 强调“人工确认后执行”，避免在设计阶段就陷入复杂的自动执行与风控闭环。

3. **风控前置**  
   任何新建套利任务都要经过风控引擎的检查，通过后才允许落地到任务和订单。

4. **收益必须分层**  
   预估收益（基于费率差）/ 已实现收益（已结算 + 已平仓）/ 未实现收益（浮动）/ 扣费后净收益（含手续费等成本）。

5. **机会与任务分表**  
   `arbitrage_opportunities`（机会快照）与 `arbitrage_tasks`（实际执行任务）严格区分，以便统计“发现了多少机会、执行了多少、成功率与兑现率”。

## 7. 后续扩展方向（V1 / V2 思路）

- 支持跨所价差套利（现货–永续、永续–永续、期货–期货等）
- 支持更多策略类型：趋势跟踪、网格、统计套利
- 完整的回测中心和参数优化（可以和现有策略复用同一引擎）
- 更细致的风险指标：VaR、最大回撤、策略相关性等
- 多用户 / 多团队协作能力（多租户）
