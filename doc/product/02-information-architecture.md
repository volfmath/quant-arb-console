# 信息架构（Information Architecture）

本文件定义 Quant Arb Console 的页面层级、导航结构和核心用户流程。  
**参考基准**：三张 UI 截图（Dashboard 概览、套利机会发现、资金费套利任务列表）。

## 1. 全局导航结构

采用**左侧图标导航 + 主内容区 + 右侧详情面板（按需）**的布局。  
**没有独立 TopBar**——系统状态、时间信息集成在侧边栏底部。

```text

│ 侧边栏 │         主内容区                  │  详情面板    │

│        │                                  │              │
│ [图标] │                                  │              │
│ [图标] │                                  │              │
│ [图标] │                                  │              │
│ [图标] │                                  │              │
│ [图标] │                                  │              │
│ [图标] │                                  │              │
│ [图标] │                                  │              │

```

侧边栏**默认图标模式（64px）**，每个菜单项只显示图标 + 图标下方小标签文字。  
选中态使用蓝色高亮背景。参照截图中的菜单结构：

## 3. 页面层级关系

```text
Level 0 ── 概览 Dashboard
│
Level 0 ── 套利机会发现
│           ├── Tab: 资金费率套利 / 价差套利
│           └── Level 1 ── 机会详情（右侧分栏面板，非覆盖式抽屉）
│                           └── 添加套利（创建任务）
│
Level 0 ── 交易监控 / 资金费套利
│           └── Level 1 ── 任务卡片详情（卡片内展开，非抽屉）
│
Level 0 ── 策略管理
│           └── Level 1 ── 策略详情 / 编辑
│
Level 0 ── 风控 / 预算监控
│           ├── Level 1 ── 风控规则编辑
│           └── Level 1 ── 告警详情
│
Level 0 ── 数据分析（收益分析 / 资产总览）
│
Level 0 ── 设置中心（各子页面）
```

**交互模式（参照截图）**：
- **套利机会页**：点击表格行 → 右侧分栏面板展开详情（主表格压缩但仍可见，非蒙层覆盖）
- **套利任务页**：每个任务是独立卡片，信息直接平铺在卡片中，无需点击展开
- **Dashboard**：所有信息平铺展示，卡片 + 图表 + 迷你表格

## 4. 核心用户流程

### 4.1 交易员主流程

```text
登录 → 概览 Dashboard
  → 查看资产总览 / 收益 / 策略运行状态
  → 进入套利机会页
    → Tab 切换：资金费率套利 / 价差套利
    → 筛选币种 / 交易所 / 最小收益
    → 点击机会行 → 右侧面板展开详情
      → 查看费率趋势图、各交易所费率排名
      → 点击"添加套利" → 填写参数
        → 风控校验通过 → 任务创建成功
  → 进入交易监控（资金费套利）页
    → 查看运行中 / 已完成 / 异常任务卡片
    → 监控每个任务的费率、仓位、保证金、收益
```

### 4.2 风控员主流程

```text
登录 → Dashboard 风控概览（半圆仪表盘 + 风险等级）
  → 查看风险敞口、杠杆使用率、预警数量
  → 进入预算监控页
    → 查看 / 处理告警
    → 调整风控规则
```

### 4.3 超级管理员主流程

```text
登录 → 设置中心
  → 配置交易所 & API Key
  → 管理用户与权限
  → 查看系统状态
```

## 5. 页面状态与数据刷新

参照截图中侧边栏底部的"自动刷新 10s"设计：

| 数据类型           | 刷新机制                   | 频率                     |
|-------------------|---------------------------|--------------------------|
| 套利机会           | 自动轮询（可配置间隔）     | 5s / 10s / 30s / 60s    |
| 资金费率           | WebSocket 推送             | 实时（~1-5s）            |
| 任务状态           | WebSocket 推送             | 实时                     |
| Dashboard KPI     | WebSocket + 轮询兜底       | 实时推送 / 30s 轮询      |
| 资产走势 / 收益图  | REST 按需                  | 切换时间范围时请求       |
| 策略运行表格       | REST 轮询                  | 30s                      |
| 风控告警           | WebSocket 推送             | 实时                     |
| 历史数据 / 报表    | REST 按需                  | 用户触发                 |

## 6. 搜索与筛选约定

参照截图中套利机会页的筛选栏设计：

1. **筛选栏**：水平排列，Dropdown + 输入框 + Toggle 组合
   - 截图示例：`[全部币种▼] [全部周期▼] [最小套利收益 >= 0.010%▼] [所有交易所▼] [仅显示正套利 🔘]`
2. **操作按钮**：筛选栏末尾提供 `[重置]` + `[高级筛选]` 按钮
3. **状态图例**：页面右上角显示状态颜色图例（如 `● 运行中  ● 已完成  ● 异常`）
4. **状态筛选**：Dropdown 选择器（如截图中的 `全部状态 ▼`）
5. **表格排序**：点击列头切换升序/降序
6. **分页**：底部居中，显示总条数 + 页码 + 每页条数选择（`10条/页 ▼`）
7. **自动刷新**：侧边栏底部可配置刷新间隔
