# MVP 路线图（MVP Roadmap）

---

## 1. 版本规划

| 版本   | 目标                         | 预计周期   | 状态   |
|--------|------------------------------|-----------|--------|
| MVP    | 资金费率套利核心闭环          | 8-10 周   | 规划中 |
| V1.0   | 完善策略 + 自动化 + 回测      | 6-8 周    | 待定   |
| V2.0   | 多策略 + 多租户 + SaaS 化     | 10-12 周  | 待定   |

---

## 2. MVP 阶段详细规划（8-10 周）

### Phase 1：基础设施（第 1-2 周）

| 任务                          | 产出                                     |
|-------------------------------|------------------------------------------|
| 项目初始化                     | 前端 React+Vite+TS 脚手架、后端 NestJS 脚手架 |
| 数据库建表                     | PostgreSQL 表结构 + Migration 文件        |
| 交易所 SDK 封装                | Binance / Bybit / OKX 统一接入层          |
| 认证鉴权                      | JWT 登录 + RBAC 权限中间件                |
| 前端 Layout 框架               | Sidebar + ContentArea + 路由骨架          |
| 深色主题基础                   | CSS Variables + Ant Design 深色配置       |

**阶段验证**：后端 lint/typecheck/unit test 通过；前端 lint/typecheck/build 通过；数据库 migration 可执行和回滚；JWT、RBAC、审计日志、WebSocket 心跳有集成测试。

### Phase 2：行情与机会（第 3-4 周）

| 任务                          | 产出                                     |
|-------------------------------|------------------------------------------|
| market-data-service           | 行情采集 + Redis 缓存                     |
| funding-rate-service          | 资金费率拉取 + 历史归档                   |
| instrument-normalizer         | 合约标准化 + unified_symbol 映射          |
| opportunity-engine            | 机会发现 + 评分 + 写入 DB                 |
| 套利机会前端页                 | Tab + KPI + 筛选 + 表格 + 详情分栏面板    |
| WebSocket 基础                | 连接管理 + opportunity:update 事件        |

**阶段验证**：mock exchange 行情与费率测试通过；合约标准化映射测试通过；机会评分固定输入输出稳定；机会 REST API 与 WebSocket 事件契约测试通过；机会页基础 E2E 通过。

### Phase 3：任务与执行（第 5-6 周）

| 任务                          | 产出                                     |
|-------------------------------|------------------------------------------|
| execution-engine              | 顺序下单 + 成交回报 + 异常补偿            |
| 任务生命周期                   | 创建 → 确认 → 执行 → 平仓 → 完成         |
| risk-engine（前置校验）        | 开仓前 6 项校验                           |
| 套利任务前端页                 | KPI 栏 + 卡片列表 + 状态差异化            |
| 创建任务流程                   | 机会详情 → 添加套利 → 风控 → 下单         |
| 订单/持仓管理                  | 前后端订单和持仓查询展示                  |

**阶段验证**：前置风控 6 项校验覆盖成功和失败用例；任务创建、暂停、恢复、停止 API 测试通过；顺序下单、单腿失败补偿、平仓状态机测试通过；所有执行测试默认使用 mock/testnet。

### Phase 4：收益与风控（第 7-8 周）

| 任务                          | 产出                                     |
|-------------------------------|------------------------------------------|
| pnl-service                   | 收益计算 + 快照 + 归因                    |
| risk-engine（运行时监控）      | 持仓中风控扫描 + 告警生成                 |
| alert-service                 | 告警持久化 + WebSocket 推送               |
| Dashboard                     | 资产总览 + 走势图 + 收益图 + 策略表 + 风控 |
| 收益分析页                     | 时间维度 + 分类 KPI + 图表 + 明细表       |

**阶段验证**：PnL 计算覆盖资金费、手续费、未实现收益、净收益；运行时风控可生成告警并推送；Dashboard 和收益分析 API 字段与页面规格一致；告警处理 E2E 通过。

### Phase 5：系统完善（第 9-10 周）

| 任务                          | 产出                                     |
|-------------------------------|------------------------------------------|
| 策略管理                       | CRUD + 启停 + 参数配置                   |
| 系统设置                       | 交易所管理 + 账户管理 + 用户管理          |
| 审计日志                       | 全操作审计 + 查询展示                     |
| 联调测试                       | 前后端联调 + 端到端测试                   |
| 性能优化                       | 大表分区 + 查询优化 + WS 节流             |
| 部署文档                       | Docker Compose + 环境配置                |

**阶段验证**：策略、系统设置、审计 API 测试通过；Docker Compose 一键启动健康检查通过；完整 E2E 跑通“登录→配置账户→查看机会→创建任务→mock 执行→收益→停止任务”；性能基准达标。

## 3. V1.0 增量规划（MVP 后 6-8 周）

| 功能                          | 说明                                     |
|-------------------------------|------------------------------------------|
| 全自动执行模式                 | 无需人工确认，策略触发后自动下单          |
| 策略参数优化                   | 基于历史数据的参数推荐                    |
| 回测中心                       | 历史数据回放 + 策略模拟                   |
| 价差套利                       | 跨所现货-合约价差套利                     |
| 通知集成                       | Telegram / 邮件 / 企业微信告警            |
| 数据导出                       | CSV / Excel 报表导出                     |
| 更多交易所                     | 接入 MEXC / Bitget 等                    |

## 4. V2.0 增量规划（V1 后 10-12 周）

| 功能                          | 说明                                     |

## 5. 技术风险与规避

| 风险                                | 影响     | 规避措施                                    |
|-------------------------------------|---------|---------------------------------------------|
| 交易所 API 频率限制                  | 高      | 请求队列 + 速率限制器 + 多 API Key 轮换     |
| 交易所 API 不稳定 / 变更            | 高      | 统一接入层隔离变化 + 健康检查 + 降级策略     |
| 双腿下单一腿失败                     | 高      | 异常补偿机制 + 实时告警 + 自动反向平仓       |
| 资金费率方向突变                     | 中      | 持仓中实时监控 + 费率反转阈值告警            |
| 数据库大表性能                       | 中      | 月度分区 + 冷热分离 + 定期归档               |
| WebSocket 大量并发推送               | 中      | 事件节流 + 批量推送 + 按页面订阅             |
